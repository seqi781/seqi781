## Repository Overview

This is a **GitHub Profile README repository** - a portfolio that displays on the user's GitHub profile page (https://github.com/seqi781). The repo name matches the username so GitHub auto-renders README.md as the profile "Overview".

## Project Structure

```
.
├── README.md    # The main GitHub profile README
└── .git/        # Git repository
```

**Important:** This repository is a profile README plus a few auto-generation workflows under `.github/`. There is no application code or test suite; "builds" are SVG/markdown regeneration on schedule.

## Purpose

The README.md serves as a personal portfolio page showcasing:
- User identity (Seqi, AI Researcher)
- Tech stack and skills
- Learning roadmap
- GitHub activity links

## Design Philosophy

The README mixes lightweight markdown with a few external SVG sources:
- demolab typing-svg for the animated header lines
- repo-hosted SVGs under `profile-3d-contrib/` (generated by `.github/workflows/profile-3d.yml`)
- snake animation served from the `output` branch (generated by `.github/workflows/snake.yml`)
- an AI weekly summary block updated by `.github/workflows/ai-weekly-summary.yml` (DeepSeek)

If a typing-svg or external badge fails to load, prefer plain markdown over adding more third-party services.

## Common Operations

### Updating the Profile
```bash
# Edit the README
vim README.md

# Commit changes
git add README.md
git commit -m "Description of changes"

# Push to GitHub (note: may need HTTP/1.1 for this repository)
git -c http.version=HTTP/1.1 push
```

### Viewing Changes
After pushing, visit https://github.com/13731890887 to see the updated profile. GitHub caches profile READMEs, so changes may take a few minutes to appear.

### Hard Refresh
If changes don't appear:
- Browser: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)
- Or add `?nocache=1` to the URL

## Known Issues

### Push Failures
Git pushes sometimes fail with "Error in the HTTP2 framing layer". Use HTTP/1.1:
```bash
git -c http.version=HTTP/1.1 push
```

### External Images
Previous versions used external image services (badgen.net, shields.io, vercel.app) which often failed to load. The current zero-dependency design avoids these issues entirely.
